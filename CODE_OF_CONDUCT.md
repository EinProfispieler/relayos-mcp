# Code of Conduct

RelayOS community spaces should be respectful, constructive, and safe for collaboration.

## Scope

This Code of Conduct applies to public RelayOS repository interactions, including issues, pull requests, discussions, and related community spaces.

## Expected behavior

- Communicate respectfully and professionally.
- Keep technical discussion constructive and evidence-based.
- Stay on topic and focused on the project.
- Respect privacy and confidentiality boundaries.
- Do not post secrets or private data.

## Unacceptable behavior

- Harassment, intimidation, or abusive language.
- Doxxing or attempts to expose personal information.
- Posting credentials, tokens, API keys, or other secrets.
- Posting confidential, proprietary, customer, or regulated data.
- Spam, repeated disruption, or bad-faith engagement.

## Enforcement

Maintainers may take actions needed to keep project spaces safe and usable, including:

- Hiding or deleting comments.
- Closing issues or pull requests.
- Limiting or blocking participation.
- Reporting abuse through platform policies and tools.

This project does not promise a formal SLA, legal process, or guaranteed response timeline for conduct reports.

## Reporting

- For security vulnerabilities, follow [SECURITY.md](SECURITY.md).
- For general conduct concerns, contact project maintainers through available repository contact paths.
- If no direct contact path is available or appropriate, use GitHub reporting tools.
