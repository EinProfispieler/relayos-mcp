# Support

This page is the primary support entry point for RelayOS Core.

Community behavior expectations are documented in [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md).

## Where to ask usage questions

- If GitHub Discussions is enabled, use Discussions for usage/help questions.
- Otherwise, open a GitHub Issue and clearly mark it as a usage question.

## Where to report bugs

- Open a GitHub Issue using the bug report template:
  - `.github/ISSUE_TEMPLATE/bug_report.md`

## Where to request features

- Open a GitHub Issue using the feature request template:
  - `.github/ISSUE_TEMPLATE/feature_request.md`

## Where to report security issues

- Follow the guidance in [SECURITY.md](SECURITY.md).
- Do not report vulnerabilities through public GitHub issues.

## Public data warning

GitHub issues and pull requests are public. Do **not** paste:

- API keys
- Tokens
- Credentials/passwords/private keys
- Private prompts
- Proprietary source code not intended for this repository
- Customer data
- Regulated data
- Logs or command output containing secrets

## RelayOS Core support boundary

- RelayOS Core is local-first, open-source, and community-supported.
- Current Core does not provide cloud sync.
- Current Core does not include telemetry by default.
- Current Core does not provide a hosted account service.
- Current Core does not include an enterprise support commitment.

## Future paid/cloud/enterprise boundary

Future paid/cloud/enterprise offerings may use separate terms, support channels, privacy notices, and consent flows.
This page describes current RelayOS Core only.
